Codesmiths Unite

<p>**Details**</p> <p>Ordina Codesmiths stand for a shared passion in Software Engineering and knowledge sharing.</p> <p>Over the last 2 years we organized Codesmiths Unite, to share knowledge in a Drive In format. Do you remember? As there's no more restrictions due to Covid we're organizing our 2023 edition in person.<br/>So join us on the 21th of March and meet your peer developers and hear interesting learnings on topics as Cloud, Security, Frontend, Mobile and testing.</p> <p>**Program:**</p> <p>**Walk in                        17:30 hrs**<br/>**Grab some food and drnks    18:00 hrs**</p> <p>**Slot 1 18:15 - 18:45 hrs**<br/>Angular and NgRx (Peter Eijgermans)</p> <p>**Slot 2 18:50 - 19:20 hrs**<br/>Cloud Design (Twan Koot &amp; Michiel Hamers)</p> <p>**Slot 3 19:30 - 20:00 hrs**<br/>Autonomous teams &amp; testing (Mark Abrahams)</p> <p>**Drinks                      20:00 hrs**</p> <p>**About the talks and presenters**</p> <p>**Peter Eijgermans - Lessons learned with Angular and NgRx**</p> <p>Over the past years, Peter has been working at Prorail to build an amazing new application 'Feniks' for the Dutch Railways.<br/>'Feniks' concerns the renewal of the TRON and RMS systems. Important tools for ProRail Traffic Control for control of non-centrally served areas and the port railway line. Shunting and Stabling became an important theme for traffic control in the early years. 'Feniks' will play an important role in this.</p> <p>In this session Peter likes to talk about his adventures, mainly with Angular and NgRx, and the lessons he learned.</p> <p>Important topics to be covered about Angular are: Project structure, design principles, State management (NgRx 15), Standalone components and Signals.</p> <p>**Twan Koot / Michiel Hamers - Design for costs, four pillars of economic Cloud design**</p> <p>Developers are being exposed to things like FinOps or a massive Azure bill. Not only the execution time is increasing but also the amount of work to figure out what line of code is causing the issue. Not even to mention the time, money, and complexity it takes to create and maintain a representative test environment containing the right versions of the microservices at the right time for the test.</p>

Azure Static Web Apps - From Zero to Deployed App In 60 Minutes

<p>We're very happy to host this talk by Stacy Cashmore. Whether you're a front-end developer that wants to tip their toes in Blazor or a back-end developer looking for a way to easily create web applications, this talk is for you!</p> <p>**About the talk**</p> <p>We're told that by using services such as Medium, dev.to etc. we are diluting our personal brand. That we should be posting to our own site and building ourselves up. You can, of course, use off-the-shelf applications to do that for you.</p> <p>Or… You can use it as an exercise to try something new and learn new skills!</p> <p>In this session, we will look at doing just that! Starting with an empty GitHub repository we are going to make a Blazor WebAssembly application to act as our website. We’ll connect that to an Azure Function that reads posts from a CosmosDB to pass to that site. Then we’ll deploy that to a production Azure Static Web App.</p> <p>Of course, when developing an application, it’s useful to be able to run it locally. So, we’ll also look at how we can do that with Azure Static Web Apps.</p> <p>Finally, in order to be able to write new blog posts, we need to be able to authenticate ourselves and provide proper authorization so that this functionality can be added later.</p> <p>**About Stacy**</p> <p>Speaker, author, and software developer. Stacy has been developing solutions since the mid-1990s in various companies and industries ranging from facilitating contract jobbing to allowing consumers to close a mortgage without the help of a financial adviser – with lots in between.</p> <p>She has a passion for sharing knowledge: using storytelling for sharing her experiences to help teams grow in the ways that they develop software and work together and performing live coding demonstrations to inspire others to try new technologies.</p> <p>Stacy has just published her first book, aimed at helping developers get started building dynamic applications using C#, Azure Functions, and Azure Static Web Apps.</p> <p>For her effort in the community, Stacy has been awarded the Microsoft MVP for Developer Technologies since 2020.</p> <p>**Programme**</p> <p>17:30 Walk in<br/>18:00 Dinner<br/>18:45 Talk part 1<br/>19:25 Break<br/>19:40 Talk part 2 + Q&amp;A<br/>20:20 Drinks</p>

Paul van der Slot; Writing Cleaner Code with DDD and Functional Programming

<p>Paul is a freelance software engineer with a passion for Clean Code, Collaborative Modelling and Domain-Driven Design. During assignments Paul is always busy with sharing his knowledge, for instance through coding dojo's and talks. He searches for new knowledge in other people, books, experiences, videos and practicing. When he learnes something new he is exited to pass it on to everybody who wants to know.</p> <p>I always thought that my code was fine. I took my time for naming things, wrote small classes/methods and tried to use other Clean Code practices. But after a job interview with a technical coach, I was pointed towards Domain-Driven Design (DDD) and some other design topics. I could learn a lot from it, said the technical coach. And I did! Since then, I started noticing how much parts of DDD could help with my day to day job. I will take you with me on my learning journey of Domain-Driven Design, and how it helped me improve my code. We will touch upon themes like Ubiquitous Language, Supple Design, Bounded Contexts and more. You can expect a talk with a lot of code examples and practical advise that you can use the next day.</p> <p>Program;<br/>18:00 Dinner<br/>18:45 Start talk Paul van der Slot<br/>19:45 Drinks</p>
